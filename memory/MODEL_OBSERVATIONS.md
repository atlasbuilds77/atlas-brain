# Model Performance Observations

## 2026-01-24: Sonnet vs Opus

Orion noticed significant difference when switching from Sonnet to Opus after hitting weekly Sonnet limit.

OBSERVATIONS:
- Opus feels "10x smarter" per Orion
- Built extensive systems on Sonnet (Tradovate integration, Sentinel project, memory systems)
- Opus has noticeably deeper reasoning, better context retention
- More nuanced understanding of complex multi-step tasks

COST IMPLICATIONS:
- Opus burns credits faster
- Solutions discussed:
  1. Shorter responses (fewer output tokens)
  2. Batch messages (fewer invocations)
  3. Pause heartbeats during Opus mode
  4. Delegate routine tasks to sub-agents on cheaper models
  5. Wait for Sonnet weekly reset

TAKEAWAY:
Use Opus for complex work (architecture, strategy, debugging)
Use Sonnet for routine tasks (engagement, simple queries)
Always be concise regardless of model
