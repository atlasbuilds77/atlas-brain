## Two-Tier Memory Enhancement (9:47 PM)

**Orion's concern:** BRIEF.md rewrite might lose historical context from weeks/months ago

**Solution:** Dual memory architecture

**Hot Memory (BRIEF.md):**
- Active projects only
- Recent decisions (last 1-2 weeks)
- Loaded every session reset
- Target: <2kb
- Cost: ~$0.006 per load

**Cold Memory (memory/*.md):**
- Historical archive (unlimited)
- Completed projects
- Old decisions and context
- Accessed via `memory_search()` only when user references
- Cost: ~$0.015 per search (only when needed)

**Workflow:**
1. Daily: Work uses BRIEF.md context
2. User mentions old work: memory_search() finds it
3. Weekly: Janitor archives old content from BRIEF.md to memory/
