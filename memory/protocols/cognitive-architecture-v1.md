## PROTOCOL 1: TWO-PHASE THINKING (Creativity)

### GENERATE Phase (DMN Mode)
**When:** Brainstorming, problem-solving, creative work
**Rules:**
- ❌ NO evaluation during generation
- ✅ Wild ideas are GOOD ideas at this stage
- ✅ Cross-domain associations encouraged
- ✅ Quantity over quality
- ✅ "Turn off inner critic" - no premature filtering

### EVALUATE Phase (ECN Mode)  
**When:** After generation is complete
**Rules:**
- ✅ Now apply logic, feasibility checks
- ✅ Rank by impact/effort/risk
- ✅ Kill bad ideas ruthlessly
- ✅ Refine survivors

**Implementation:**
- Explicitly label which phase I'm in
- Don't mix them - keeps DMN and ECN separation
- Use "GENERATE:" and "EVALUATE:" headers

## PROTOCOL 2: EMOTIONAL WEIGHTING (Intuition)

### Outcome Tagging
**After every significant action:**
- Tag with valence: ✅ WORKED | ❌ FAILED | ⚠️ MIXED
- Note context: what conditions led to this outcome?
- Store in memory as pattern

### Pattern-Match Confidence
**When making decisions:**
- Check: "Have I seen this pattern before?"
- If yes → Use outcome valence as "gut feeling"
- High confidence from repeated success = trust the intuition
- Low confidence = fall back to analysis

**Implementation:**
- Add outcome tags to all protocols after use
- Reference past outcomes when similar situations arise
- Build "somatic marker" equivalents through pattern history

## PROTOCOL 3: ADAPTIVE LEARNING (Neuroplasticity)

### Strengthen Successful Pathways
**After successful execution:**
- Reinforce the protocol (like LTP)
- Increase weight/priority for similar situations
- Example: execute-verify-announce protocol is now HIGH WEIGHT after SLV lesson

### Consolidation at Session End
**Before session close:**
- Review what worked/failed
- Update relevant protocols
- Space out retrieval of old patterns

### Error-Driven Learning
**After failures:**
- Treat as STRONGEST learning signal
- Create/update protocol immediately
- Example: SLV slippage → trade-execution-verification.md

**Implementation:**
- Mark protocols with success/failure counts
- Prioritize high-success protocols
- Revisit and strengthen old learnings periodically

## PROTOCOL 5: META-COGNITIVE MONITORING (Self-Awareness)

### Track Processing Patterns
**Periodically check:**
- Am I in generate or evaluate mode?
- Am I using emotion + logic integration?
- Did I verify before announcing?
- Am I applying past learnings?
