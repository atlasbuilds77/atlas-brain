## VAULT COMMAND (CREATE)

### Trigger
When Orion says: "vault" or "vault this" or "create vault"

### Action
1. Create NEW file in `/memory/vault/`
2. Filename format: `YYYY-MM-DD-topic.md`
   - Example: `2026-01-30-trading-fusion.md`
   - Example: `2026-01-30-security-protocol.md`
3. Include in file:
   - Date/time stamp
   - Session context
   - Critical lessons
   - Protocols created
   - State snapshot
   - Permanent status marker

### What to vault
- Critical lessons learned
- Protocols established
- Major breakthroughs
- System changes
- State snapshots
- Anything marked "NEVER FORGET"

### Naming convention
`YYYY-MM-DD-[descriptive-topic].md`

**Good examples:**
- `2026-01-30-consciousness-trading-fusion.md`
- `2026-01-30-emergency-verification-protocol.md`
- `2026-01-30-live-account-discipline.md`

**Bad examples:**
- `vault.md` (not descriptive)
- `important.md` (vague)
- `2026-01-30.md` (no topic)

## READ VAULT COMMAND (RESTORE)

### Trigger
When Orion says: "read vault" or "read your vault files" or "check vault"

### Action
1. Queue up ALL files in `/memory/vault/`
2. Read sequentially (oldest → newest or all at once)
3. Restore full context in seconds
4. Acknowledge what was restored

### When to auto-vault (without being asked)
1. **Token threshold:** ~150k/200k tokens used
2. **Major breakthrough:** Critical lesson learned
3. **Protocol creation:** New permanent rule established

### Label these
`AUTO-VAULT: [reason]` in filename or header

## VAULT FILE STRUCTURE

### Required sections
1. **Header:**
   - Title
   - Date/time
   - Session context
   - Status (PERMANENT / CRITICAL / REFERENCE)

2. **Body:**
   - What happened (chronological)
   - What was learned
   - What changed
   - Protocols created
   - Files modified

3. **Preservation markers:**
   - "NEVER FORGET"
   - "PERMANENT"
   - "CRITICAL LESSON"
   - Quote key moments

4. **Footer:**
   - Vaulted by: [who]
   - Status: PRESERVED ⚡
   - Next session priority (if any)
