# Real Node Mapping System for Atlas Brain

This system maps actual memory files to brain nodes in a Three.js visualization, with automatic detection of new files and spatial positioning based on node types.

## Files Created

1. **`real-brain-nodes.js`** - Core module with all functionality
2. **`test-real-nodes.html`** - Complete test/demo page
3. **`integration-example.js`** - Integration helper for existing Three.js scenes
4. **`README-real-nodes.md`** - This documentation

## Core Features

### 1. **Automatic Node Detection**
- Reads from `/tmp/real-brain-nodes.json` (generated by the brain system)
- Maps 89 memory files + 6 tools + 5 cognitive systems
- Auto-rescans every 10 seconds for new files

### 2. **Spatial Positioning**
Nodes are placed in brain-appropriate regions:
- **Protocols (cyan)** - Front left hemisphere
- **Trading (blue)** - Front right hemisphere  
- **People (yellow)** - Temporal lobes
- **Tools (purple)** - Motor cortex area
- **Cognitive systems** - Distributed (orange/red/green/pink)

### 3. **Node Animations**
- **Fade-in** when nodes first appear
- **Scale pulse** for active visualization
- **Gentle floating** movement
- **Glow effects** with emissive materials

### 4. **Real-time Updates**
- Polls for new `.md` files every 10 seconds
- Spawns new nodes automatically
- Maintains node state tracking

## Quick Start

### Option 1: Test Page
Open `test-real-nodes.html` in a browser to see the complete visualization.

### Option 2: Integration into Existing Three.js Scene

```javascript
import { integrateRealNodes } from './real-brain-nodes.js';

// Initialize with your Three.js scene
const nodeIntegration = await integrateRealNodes(yourScene, {
  nodeScale: 0.2,
  glowEnabled: true,
  autoUpdate: true
});

// In your animation loop
function animate(time) {
  const deltaTime = (time - lastTime) / 1000;
  nodeIntegration.update(deltaTime);
  // ... rest of your animation code
}
```

### Option 3: Direct API Usage

```javascript
import { 
  loadRealNodes, 
  watchForNewNodes, 
  updateNodeAnimations,
  getNodeCounts 
} from './real-brain-nodes.js';

// Load nodes
const nodes = await loadRealNodes();

// Watch for new nodes
const stopWatching = watchForNewNodes((newNodes) => {
  console.log('New nodes detected:', newNodes);
  // Add to your visualization
});

// Update animations in your loop
updateNodeAnimations(nodes, deltaTime);

// Get statistics
const counts = getNodeCounts();
console.log(`Total nodes: ${counts.total}`);
```

## API Reference

### `loadRealNodes()`
```javascript
async function loadRealNodes(): Promise<Node[]>
```
Loads and processes node data from `/tmp/real-brain-nodes.json`. Returns an array of positioned node objects.

### `watchForNewNodes(callback, intervalMs = 10000)`
```javascript
function watchForNewNodes(
  callback: (newNodes: Node[]) => void,
  intervalMs?: number
): () => void
```
Starts polling for new nodes. Returns a function to stop watching.

### `updateNodeAnimations(nodes, deltaTime)`
```javascript
function updateNodeAnimations(
  nodes: Node[],
  deltaTime: number
): void
```
Updates animation states for all nodes. Call this in your animation loop.

### `getNodeCounts()`
```javascript
function getNodeCounts(): {
  protocols: number,
  trading: number,
  people: number,
  tools: number,
  cognitive: number,
  total: number
}
```
Returns current node counts by type.

### `findNodes(query)`
```javascript
function findNodes(query: string): Node[]
```
Searches for nodes by name or file path.

## Node Data Structure

Each node object contains:
```javascript
{
  id: string,                    // Unique identifier
  name: string,                  // Node name (file name without .md)
  type: string,                  // 'protocols', 'trading', 'people', 'tools', 'cognitive'
  color: number,                 // THREE.js color integer
  position: { x, y, z },        // 3D position
  scale: number,                 // Current scale (animated)
  opacity: number,              // Current opacity (animated)
  animation: {                  // Animation state
    birthTime: number,
    scalePulse: number,
    fadeProgress: number
  },
  metadata: {                   // Original data
    file: string,               // File path
    region: string,             // Region identifier
    colorName: string           // Original color name
  }
}
```

## Color Mapping

- `cyan` (0x00ffff) - Protocols
- `blue` (0x0077ff) - Trading
- `yellow` (0xffff00) - People
- `purple` (0xaa00ff) - Tools
- `orange` (0xff8800) - Pattern detection
- `red` (0xff4444) - Emotional intelligence
- `green` (0x44ff44) - Metacognition
- `pink` (0xff66cc) - Bias detection

## Spatial Positioning Logic

Nodes are positioned using spherical coordinates within their designated brain regions:

1. **Protocols**: Front left hemisphere (`x: -2.5, y: 1.5, z: 1.5`)
2. **Trading**: Front right hemisphere (`x: 2.5, y: 1.5, z: 1.5`)
3. **People**: Temporal lobes (`x: 0, y: 0, z: -1.5`)
4. **Tools**: Motor cortex (`x: 0, y: -1.5, z: 0.5`)
5. **Cognitive**: Distributed based on function type

## Troubleshooting

### 1. JSON File Not Found
The system will generate fallback nodes if `/tmp/real-brain-nodes.json` is not accessible.

### 2. CORS Issues
If running from a web server, ensure the JSON file is served with appropriate CORS headers, or use a local development server.

### 3. Performance
For large numbers of nodes, consider:
- Using instanced meshes
- Reducing geometry complexity
- Limiting glow effects

## Extending the System

### Adding New Node Types
1. Update `COLOR_MAP` with new colors
2. Add positioning in `POSITION_CONFIG`
3. Update processing logic in `processNodeData()`

### Custom Animations
Override `updateNodeAnimations()` or extend the integration class to add custom animation behaviors.

### Custom Styling
Modify `createNodeMesh()` to use different geometries, materials, or effects.
