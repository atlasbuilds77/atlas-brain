# Meridian Scanner Bug Fix - 2026-02-27

## Issue Report
**Symptom:** Scanner appeared "stuck" after logging "Starting market-open sweep scanner..." with no subsequent activity visible in logs.

**Location:** `meridian_scanner.py` line 341+ in `run_market_scan()` while loop

## Root Cause Analysis
The while loop was **NOT actually stuck** - it was executing correctly but had no visibility logging between the initial "Starting..." message and sweep detection events. This created the illusion that the loop wasn't running.

**Evidence:**
1. Testing confirmed the loop executes properly - fetches bars, checks for sweeps, sleeps 15s between iterations
2. PM levels were being fetched successfully ("Late PM levels: H=609.85 L=602.51")
3. No errors in fetch_bars() - API calls returning 200 with valid data
4. Loop logic and timing checks all working as designed

## Fix Applied
Added **heartbeat logging** to provide visibility into scanner activity:

### Changes Made:

1. **Added iteration counter to ScannerState** (line 46):
```python
scan_iterations: int = 0  # Track loop iterations for heartbeat logging
```

2. **Added heartbeat log in main loop** (line 394-398):
```python
self.state.scan_iterations += 1

# Heartbeat log on first iteration and then every 8 iterations (~2 minutes at 15s interval)
if self.state.scan_iterations == 1 or self.state.scan_iterations % 8 == 0:
    close_price = float(latest.get("close", latest.get("price", 0)))
    log.info(f"Scanning... iter={self.state.scan_iterations}, bar={self.state.bar_count}, price={close_price:.2f}, PM H={self.state.pm_high:.2f}/L={self.state.pm_low:.2f}")
```
