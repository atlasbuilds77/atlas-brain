## Daily Workflow Template

### Phase 1: Morning Prep (15 minutes)
**Atlas can help with:**
- Prompt interpretation and brainstorming
- Research direction and examples
- Theme selection for the day

**Tasks:**
1. Check email for Daily UI prompt
2. Quick research on Collect UI for that prompt number
3. Brainstorm 3 different approaches
4. Select a theme/context (e.g., "Fitness app calculator")

### Phase 2: Design Execution (45-60 minutes)
**Atlas can help with:**
- Component suggestions and patterns
- Color palette recommendations
- Typography scale setup
- Design principle reminders

**Tasks:**
1. Set up artboard with grid (8px baseline)
2. Establish color palette (primary, secondary, accent)
3. Define typography scale (H1, H2, Body, Caption)
4. Create wireframe/layout
5. Build high-fidelity design
6. Add interaction states (hover, active, disabled)

### Phase 3: Polish & Document (30 minutes)
**Atlas can help with:**
- Writing design rationale
- Accessibility checking
- Presentation formatting
- Tag/hashtag suggestions

**Tasks:**
1. Add annotations explaining design decisions
2. Check color contrast ratios
3. Create presentation view (clean background, device mockup)
4. Write 2-3 sentence rationale
5. Export for sharing (PNG, 2x resolution)

### Phase 4: Share & Learn (15 minutes)
**Atlas can help with:**
- Community engagement suggestions
- Feedback analysis
- Progress tracking

**Tasks:**
1. Post to Dribbble/Behance with #dailyui #[promptnumber]
2. Share on LinkedIn/Twitter with learning insights
3. Engage with 2-3 other Daily UI submissions
4. Log progress in tracking sheet

## Common Prompt Categories & Strategies

### Form-Based Prompts (Sign Up, Contact Us, File Upload)
**Atlas Support:**
- Form best practices checklist
- Validation pattern examples
- Progressive disclosure strategies

**Strategy:**
1. Identify required vs optional fields
2. Plan validation and error states
3. Consider multi-step flows for complex forms

### Dashboard Prompts (Analytics, Leaderboard, Monitoring)
**Atlas Support:**
- Data visualization principles
- Information hierarchy templates
- Real-time update patterns

**Strategy:**
1. Define primary metrics
2. Establish visual hierarchy
3. Plan empty/loading/error states

### Commerce Prompts (Checkout, Shopping Cart, Pricing)
**Atlas Support:**
- Conversion optimization tips
- Trust-building elements
- Mobile commerce patterns

**Strategy:**
1. Simplify checkout flow
2. Build trust signals
3. Optimize for mobile

### Content Prompts (Blog, News, Testimonials)
**Atlas Support:**
- Content hierarchy templates
- Reading experience optimization
- Social proof patterns

**Strategy:**
1. Prioritize readability
2. Establish content rhythm
3. Add social engagement elements
